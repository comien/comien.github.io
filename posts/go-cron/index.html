go-cron 源码分析

Posted by 雾雨七辰 on Wed, Jul 6, 2022

go-cron 是我们开发中经常使用的定时任务库

1. 基本使用

1.1 install

1$ go get github.com/robfig/cron/v3@v3.0.0

1.2 直接传入 func 的方式

 1func main() {
 2	// 如果上次任务尚未完成就跳过执行
 3	task := cron.New(cron.WithSeconds(), cron.WithChain(cron.SkipIfStillRunning(cron.VerbosePrintfLogger(log.New(os.Stdout, "cron: ", log.LstdFlags)))))
 4  // 每2秒执行一次
 5	task.AddFunc("*/2 * * * * ?", func() {
 6		log.Println("任务")
 7	})
 8	// 启动计划任务
 9	task.Start()
10	// 不能关闭已经在执行中的任务.
11	defer task.Stop()
12	select {}
13}

1.3 使用 struct 的方式

 1// 定义struct
 2type Task struct {
 3}
 4// 为其实现函数 Run()
 5func (t Task) Run() {
 6	log.Println("任务")
 7}
 8
 9func main() {
10	// 上次任务未完成下次跳过
11	task := cron.New(cron.WithSeconds(), cron.WithChain(cron.SkipIfStillRunning(cron.VerbosePrintfLogger(log.New(os.Stdout, "cron: ", log.LstdFlags)))))
12	task.AddJob("*/2 * * * * ?", Task{}) // 添加 job
13	// 启动计划任务
14	task.Start()
15	// 不能关闭已经在执行中的任务.
16	defer task.Stop()
17	select {}
18}

2. 源码分析

2.1 结构体定义

 1type Cron struct {
 2	entries   []*Entry
 3	chain     Chain
 4	stop      chan struct{}
 5	add       chan *Entry
 6	remove    chan EntryID
 7	snapshot  chan chan []Entry
 8	running   bool
 9	logger    Logger
10	runningMu sync.Mutex
11	location  *time.Location
12	parser    Parser
13	nextID    EntryID
14	jobWaiter sync.WaitGroup
15}

2.2 AddFunc

 1type FuncJob func()
 2
 3func (f FuncJob) Run() { f() }
 4
 5// AddFunc adds a func to the Cron to be run on the given schedule.
 6// The spec is parsed using the time zone of this Cron instance as the default.
 7// An opaque ID is returned that can be used to later remove it.
 8func (c *Cron) AddFunc(spec string, cmd func()) (EntryID, error) {
 9	return c.AddJob(spec, FuncJob(cmd))
10}

可以看到，AddFunc 中也是调用的 AddJob，AddFunc 将传入的 func 包装为 FuncJob 类型，并且 FuncJob 实现了函数 Run()

2.3 AddJob

 1// AddJob adds a Job to the Cron to be run on the given schedule.
 2// The spec is parsed using the time zone of this Cron instance as the default.
 3// An opaque ID is returned that can be used to later remove it.
 4func (c *Cron) AddJob(spec string, cmd Job) (EntryID, error) {
 5	schedule, err := c.parser.Parse(spec)
 6	if err != nil {
 7		return 0, err
 8	}
 9	return c.Schedule(schedule, cmd), nil
10}

AddJob 调用了解析器，解析传入的 cron 表达式（我们这里不具体分析 cron 表达式的解析），得到 Schedule type struct 接下来调用 Schedule 方法

2.4 Schedule

 1// Schedule adds a Job to the Cron to be run on the given schedule.
 2// The job is wrapped with the configured Chain.
 3func (c *Cron) Schedule(schedule Schedule, cmd Job) EntryID {
 4	c.runningMu.Lock()
 5	defer c.runningMu.Unlock()
 6	c.nextID++
 7	entry := &Entry{
 8		ID:         c.nextID,
 9		Schedule:   schedule,
10		WrappedJob: c.chain.Then(cmd),
11		Job:        cmd,
12	}
13	if !c.running {
14		c.entries = append(c.entries, entry)
15	} else {
16		c.add <- entry
17	}
18	return entry.ID
19}

在 Schedule 中，通过加锁保证并发安全，获取一个任务 id，构建了 Entry struct
此时判断了任务状态

  • 如果 cron 不是 running status, append 到 entries 数组中
  • 如果 cron 是 running status, 就放到 channel 中

2.5 Start

 1// Start the cron scheduler in its own goroutine, or no-op if already started.
 2func (c *Cron) Start() {
 3	c.runningMu.Lock()
 4	defer c.runningMu.Unlock()
 5	if c.running {
 6		return
 7	}
 8	c.running = true
 9	go c.run()
10}

接下来，在 Start 方法中，可以看到，更新了 running status，并通过协程的方式，调用私有方法 run()
因为方法 run() 是通过协程调用的，因此这里不会阻塞，所以调用完成 Start 后，需要我们保证主线程不会退出
然后我们继续跟进方法 run()

2.6 run

run() 方法内容比较多，我们分几段分析

2.6.1 遍历 entries

 1func (c *Cron) run() {
 2	c.logger.Info("start")
 3
 4	// Figure out the next activation times for each entry.
 5	now := c.now()
 6	for _, entry := range c.entries {
 7		entry.Next = entry.Schedule.Next(now)
 8		c.logger.Info("schedule", "now", now, "entry", entry.ID, "next", entry.Next)
 9	}
10
11	// --- snip ---
12}

首先进入 run() 后，获取了当前时间，并对 entries 进行了遍历
通过每个 Entry 中的 Schedule 计算下次执行时间，存放在 entry.Next 字段

2.6.2 排序并产生一个定时触发器

 1// --- snip ---
 2//  时间排序实现
 3type byTime []*Entry
 4
 5func (s byTime) Len() int      { return len(s) }
 6func (s byTime) Swap(i, j int) { s[i], s[j] = s[j], s[i] }
 7func (s byTime) Less(i, j int) bool {
 8	// Two zero times should return false.
 9	// Otherwise, zero is "greater" than any other time.
10	// (To sort it at the end of the list.)
11	if s[i].Next.IsZero() {
12		return false
13	}
14	if s[j].Next.IsZero() {
15		return true
16	}
17	return s[i].Next.Before(s[j].Next)
18}
19
20// --- snip ---
21func (c *Cron) run() {
22	// --- snip ---
23
24	for {
25		// Determine the next entry to run.
26		sort.Sort(byTime(c.entries))
27
28		var timer *time.Timer
29		if len(c.entries) == 0 || c.entries[0].Next.IsZero() {
30			// If there are no entries yet, just sleep - it still handles new entries
31			// and stop requests.
32			timer = time.NewTimer(100000 * time.Hour)
33		} else {
34			timer = time.NewTimer(c.entries[0].Next.Sub(now)) // 定时触发器
35		}
36
37		// --- snip ---
38	}
39}

entries 完成遍历后，run() 进入死循环
循环内

  • 首先对 entries 进行排序，排序方式是通过 Next 字段的比较
  • 然后获取第一个任务，计算第一个任务和当前时间差，生成定时触发器（如果没有任务，触发器默认了一个很长的时间）

2.6.3 轮询触发器

 1func (c *Cron) run() {
 2	// --- snip --- 
 3
 4		for {
 5			select {
 6			case now = <-timer.C:
 7				now = now.In(c.location)
 8				c.logger.Info("wake", "now", now)
 9
10				// Run every entry whose next time was less than now
11				for _, e := range c.entries {
12					if e.Next.After(now) || e.Next.IsZero() {
13						break // 因为 entries 已经做了排序，所有如果某个任务没到时间，就结束遍历
14					}
15					c.startJob(e.WrappedJob)
16					e.Prev = e.Next
17					e.Next = e.Schedule.Next(now)
18					c.logger.Info("run", "now", now, "entry", e.ID, "next", e.Next)
19				}
20
21			// --- snip ---
22
23			break
24		}
25	}
26}

最后 run() 进行入轮询
当任务时间到了，定时触发器 channel 得到一个时间，将时间转换成 time.Local
遍历所有的 entries
因为 entries 已经做了排序，所有如果某个任务没到时间，那剩余的任务也没到时间，就结束遍历
到达时间的任务调用 startJob

1func (c *Cron) startJob(j Job) {
2	c.jobWaiter.Add(1)
3	go func() {
4		defer c.jobWaiter.Done()
5		j.Run()
6	}()
7}

startJob 启动一个协程跑结构体实现的函数 Run()

1case newEntry := <-c.add:
2				timer.Stop()
3				now = c.now()
4				newEntry.Next = newEntry.Schedule.Next(now)
5				c.entries = append(c.entries, newEntry)
6				c.logger.Info("added", "now", now, "entry", newEntry.ID, "next", newEntry.Next)

继续回到轮询逻辑
run() 也会轮询 add channel，前边提到过，如果 cron 在 running status，新添加的任务就是放到 add channel 中
所以这里拿到新增的任务后，停止计时器，计算任务的下次执行时间，添加到 entries 中，下次死循环逻辑中这个任务就生效了
到这里基本上就结束了，剩下就是 cron 快照、停止、删除任务的逻辑，逻辑不复杂，这里不再具体分析
